# Glossary

Terms used in this project.

## OAuth / OIDC

| Term                                   | Definition                                                                                                                                                                                           |
| -------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **OAuth 2.0**                          | Authorization framework (RFC 6749). Allows a client to access resources on behalf of a user without sharing credentials. Being consolidated into OAuth 2.1 (IETF draft).                             |
| **OIDC** (OpenID Connect)              | Identity layer on top of OAuth 2.0. Adds authentication (who the user is) to OAuth's authorization (what the user allows).                                                                           |
| **Authorization Code Flow**            | OAuth grant type where the client receives a short-lived code, then exchanges it for tokens via a back-channel request.                                                                              |
| **PKCE** (Proof Key for Code Exchange) | Extension (RFC 7636) that binds the authorization request to the token request using a `code_verifier`/`code_challenge` pair. Prevents code interception. Mandatory for all clients under OAuth 2.1. |
| **code_verifier**                      | A random string (43-128 chars) generated by the client. Sent to the token endpoint during code exchange. Never sent to the authorization endpoint.                                                   |
| **code_challenge**                     | `BASE64URL(SHA-256(code_verifier))`. Sent to the authorization endpoint. The server stores it and later verifies it against the `code_verifier`.                                                     |
| **Authorization Endpoint**             | Server endpoint (`/api/auth/authorize`) where the user authenticates and consents. Returns an authorization code.                                                                                    |
| **Token Endpoint**                     | Server endpoint (`/api/auth/token`) that exchanges an authorization code for tokens. Accepts `application/x-www-form-urlencoded`.                                                                    |
| **UserInfo Endpoint**                  | OIDC endpoint (`/api/auth/userinfo`) that returns user claims given a valid access token.                                                                                                            |
| **Discovery Endpoint**                 | `/.well-known/openid-configuration`. Returns server metadata (supported endpoints, scopes, algorithms). Defined by OIDC Discovery 1.0 (RFC 8414 covers OAuth-only metadata at a different path).     |
| **access_token**                       | Opaque token used to call protected APIs (e.g., `/api/auth/userinfo`). Sent as `Authorization: Bearer <token>`.                                                                                      |
| **id_token**                           | JWT containing user identity claims (sub, email, name). Issued alongside the access_token in OIDC flows.                                                                                             |
| **JWT** (JSON Web Token)               | Compact token format: `header.payload.signature` (Base64URL-encoded). OIDC default signing is RS256; this demo uses HS256 for simplicity.                                                            |
| **state**                              | Random value sent in the authorization request and returned in the callback. Client validates it matches to prevent CSRF attacks.                                                                    |
| **redirect_uri**                       | URL where the authorization server sends the user after consent. Must match between the authorize and token requests.                                                                                |
| **client_id**                          | Identifier for the client application (`notifoo`). Sent in both the authorize and token requests.                                                                                                    |
| **grant_type**                         | Specifies which OAuth flow is being used. This project uses `authorization_code`.                                                                                                                    |
| **scope**                              | Permissions requested by the client. `openid profile email` requests identity, profile info, and email address.                                                                                      |
| **public client**                      | OAuth client that cannot store secrets (SPAs, PWAs, mobile apps). Must use PKCE. Under OAuth 2.1, even confidential clients must use PKCE.                                                           |
| **consent page**                       | UI where the user reviews and approves the requested permissions before the authorization code is issued.                                                                                            |

## PWA

| Term                          | Definition                                                                                                                     |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| **PWA** (Progressive Web App) | Web application that uses Service Workers, a manifest, and HTTPS to provide native-app-like features (offline, install, push). |
| **Service Worker**            | Background script that intercepts network requests. Enables offline caching and push notifications.                            |
| **Web App Manifest**          | JSON file (`manifest.json`) that describes the app (name, icons, display mode) for install and home screen.                    |
| **Workbox**                   | Google library for Service Worker caching strategies. This project uses `precacheAndRoute` and `NavigationRoute`.              |
| **precacheAndRoute**          | Workbox method that caches listed assets at install time and serves them from cache.                                           |
| **NavigationRoute**           | Workbox route that serves the app shell (`/index.html`) for all navigation requests (SPA routing).                             |
| **standalone**                | Display mode where the PWA runs in its own window without browser UI (address bar, tabs).                                      |
| **window-controls-overlay**   | Display mode where the PWA integrates with the OS title bar for a more native look.                                            |
| **display_override**          | Manifest field that specifies preferred display modes in priority order.                                                       |
| **maskable icon**             | Icon with a safe zone that the OS can crop into any shape (circle, rounded square). Required for adaptive icons on Android.    |
| **form_factor**               | Manifest screenshot field (`wide` or `narrow`) that tells the install UI which device type the screenshot represents.          |
| **beforeinstallprompt**       | Browser event fired when the PWA meets installability criteria. Can be captured to show a custom install button.               |

## Web Awesome

| Term               | Definition                                                                                                              |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| **Web Awesome**    | UI component library (successor to Shoelace). Provides `wa-button`, `wa-card`, etc. as custom elements with Shadow DOM. |
| **Shadow DOM**     | Browser feature that encapsulates a component's DOM and styles, preventing external CSS from leaking in.                |
| **Custom Element** | W3C standard for defining new HTML tags (e.g., `<wa-button>`). Registered via `customElements.define()`.                |

## Infrastructure

| Term                                                    | Definition                                                                                                                               |
| ------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **GCS** (Google Cloud Storage)                          | Object storage. Hosts the PWA's static files.                                                                                            |
| **Cloud CDN**                                           | Google's content delivery network. Caches static assets at edge locations.                                                               |
| **Cloud Run**                                           | Serverless container platform. Runs the push notification + OAuth demo server.                                                           |
| **Global External Application LB**                      | Layer 7 load balancer (`EXTERNAL_MANAGED`). Routes traffic, terminates SSL, serves as CDN origin.                                        |
| **sslip.io**                                            | Free DNS service that maps `<IP>.sslip.io` to that IP. Used for HTTPS without a custom domain.                                           |
| **OpenTofu**                                            | Open-source infrastructure-as-code tool (Terraform fork). Manages all GCP resources in `tofu/`.                                          |
| **VAPID** (Voluntary Application Server Identification) | Key pair used to authenticate push notification servers. The public key is shared with the browser; the private key signs push messages. |
| **Artifact Registry**                                   | Google's container image registry. Stores Docker images for Cloud Run deployment.                                                        |

## Tooling

| Term             | Definition                                                                                        |
| ---------------- | ------------------------------------------------------------------------------------------------- |
| **VitePlus**     | Unified frontend toolchain wrapping Vite 8, Rolldown, Oxlint, Oxfmt. CLI is `vp`.                 |
| **Rolldown**     | Rust-based bundler used by VitePlus (replaces both Rollup and esbuild as Vite's unified bundler). |
| **Oxlint**       | Rust-based linter used by VitePlus (replaces ESLint).                                             |
| **Oxfmt**        | Rust-based formatter used by VitePlus (replaces Prettier).                                        |
| **bun**          | JavaScript runtime and package manager. Used for dependency management and running scripts.       |
| **Playwright**   | E2E testing framework. Runs browser tests against the built PWA.                                  |
| **oauth4webapi** | Standards-compliant OAuth 2.0 / OIDC client library (by panva). Used for RFC conformance testing. |
| **Hono**         | Lightweight web framework for edge/serverless. Runs the push + OAuth demo server.                 |
